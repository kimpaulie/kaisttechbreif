# 시간 컨텍스트 결정 룰 (F1)

SKILL.md `[1] 시간 컨텍스트 확정` 단계의 세부 룰.

## 1. 시각 기준

- **항상 KST(UTC+9)** 기준.
- 사용자 환경 시간이 다른 타임존이어도 KST로 변환해 판단한다.

## 2. 발행 일자(id) 결정

발행되는 JSON 파일의 `id`는 **"오늘"** 일자 (스킬을 실행한 그 날의 KST 날짜).
다루는 "기간"은 별도로 결정한다 (아래).

## 3. 다루는 기간(coverage) 결정 매트릭스

| 오늘 요일 | 기본 커버리지 | 비고 |
|---|---|---|
| 화 | 어제(월) 하루 | 평일 표준 |
| 수 | 어제(화) 하루 | 평일 표준 |
| 목 | 어제(수) 하루 | 평일 표준 |
| 금 | 어제(목) 하루 | 평일 표준 |
| 토 | 어제(금) 하루 | 평일 표준, 단 한국 매체는 보도량 줄어듦 |
| 일 | 토 하루 | 한국 매체 거의 휴무 → 글로벌 보도 비중 자연스레 ↑ |
| **월** | **금/토/일 3일 묶음** | 주말 묶음 처리 |

## 4. 공휴일 조정

- 사용자 발화에 공휴일 정보가 명시되면 그것을 신뢰한다.
  - 예: "어제(부처님오신날)는 보도가 없을 텐데..." → 휴일+직전 평일까지 묶음.
- 명시 없고 일반적인 공휴일 위치(설/추석/광복절 등 잘 알려진 날짜)면 직전 평일까지 자동 확장.
- 불확실하면 묶지 않고 표준 룰로 진행.

## 5. 사용자 안내 멘트 (한 줄)

기간 결정 결과는 반드시 사용자에게 한 줄로 알린다. 예:

- 평일: `"어제(5/26 화요일) 보도를 정리하겠습니다."`
- 월요일: `"오늘은 월요일이라 금/토/일(5/23~5/25) 묶어서 정리하겠습니다."`
- 공휴일: `"5/24~5/25는 부처님오신날 연휴라 5/22 금요일까지 묶어서 보겠습니다."`

## 6. dayOfWeek 매핑

JSON 출력의 `dayOfWeek` 필드는 **오늘**(발행 일자)의 요일을 한 글자로:

| JS getDay() | 한 글자 |
|---|---|
| 0 | 일 |
| 1 | 월 |
| 2 | 화 |
| 3 | 수 |
| 4 | 목 |
| 5 | 금 |
| 6 | 토 |

## 7. 왜 이렇게 분기하나

v7~v8 회고에서 발견된 가장 큰 빈틈이 일요일/공휴일 처리였다. 한국 매체가 주말·휴일에는 거의 안 돌아가므로, 단일 날짜만 보면 후보가 부족하고 한국어 보도 게이트에서 대거 탈락한다. **묶음 기간**으로 보정하면 다양성·신선도를 유지할 수 있다.
