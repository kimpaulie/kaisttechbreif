# 시간 컨텍스트 결정 룰 (F1)

SKILL.md `[1] 시간 컨텍스트 확정` 단계의 세부 룰. 발행 일자와 다루는 기간을 정하고, 사이트에 표시될 `coverage.label`까지 만들어 낸다.

## 1. 시각 기준

- **항상 KST(UTC+9)**.
- 한국 공휴일 데이터는 `.claude/skills/tech-news-daily/data/kr-holidays.json` 참조.
- 2027년까지 미리 박혀 있고, 그 이후는 매년 갱신 필요.

## 2. 발행 일자(`id`) 결정

발행되는 JSON 파일의 `id`는 **오늘** 일자(KST). 즉, 스킬을 실행하는 그 날.

**예외**: 오늘이 공휴일(또는 토·일) → **발행 스킵**. 사용자에게 다음과 같이 알린다:

> "오늘(YYYY-MM-DD)은 [공휴일명/주말]이라 발행을 건너뛰겠습니다. 다음 영업일에 다시 실행해 주세요."

(자동 발행 자체를 영업일에만 돌리는 것이 정상이지만, 안전망으로 가드 둠.)

## 3. 다루는 기간(`coverage`) 결정 매트릭스

**핵심 규칙**: 다루는 기간은 "발행 직전 영업일의 다음 시점부터 어제까지" 모두 포함.

| 오늘 (영업일) | 다루는 기간 | 비고 |
|---|---|---|
| 화 (일반) | 월 | 단일 평일 |
| 수 | 화 | 단일 평일 |
| 목 | 수 | 단일 평일 |
| 금 | 목 | 단일 평일 |
| 월 | 금/토/일 (3일) | 주말 묶음 (기존) |
| 공휴일 직후 영업일 | 직전 영업일 끝부터 어제까지 모두 포함 | 아래 알고리즘 참조 |

### 알고리즘 (의사코드)

```
today = KST 오늘 일자
if today in 공휴일 or today.요일 in [토, 일]:
    "발행 스킵" 안내 후 종료

# 직전 영업일 찾기 (오늘 전날부터 거꾸로)
prev_business = today - 1일
while prev_business in 공휴일 or prev_business.요일 in [토, 일]:
    prev_business -= 1일

# 직전 영업일 발행이 이미 prev_business까지 커버했다고 가정
# → 이번 발행이 다루는 기간 = (prev_business + 1일) ~ (today - 1일)
coverage.from = prev_business + 1일
coverage.to   = today - 1일

# 단일 일자면 from == to
```

### 예시

| 오늘 | prev 영업일 | coverage.from | coverage.to | label |
|---|---|---|---|---|
| 2026-05-26 화 | 2026-05-22 금 (월 어린이날대체·화 부처님오신날대체 가정) | 2026-05-23 | 2026-05-25 | `"5월 23~25일 (토·일·월)"` |
| 2026-05-27 수 | 2026-05-26 화 | 2026-05-26 | 2026-05-26 | `"5월 26일 (화)"` |
| 2026-03-03 화 | 2026-02-27 금 (3/1 삼일절 일 + 3/2 대체월) | 2026-02-28 | 2026-03-02 | `"2월 28일~3월 2일 (토·일·월)"` |
| 2026-09-28 월 | 2026-09-23 수 (9/24~26 추석) | 2026-09-24 | 2026-09-27 | `"9월 24~27일 (목·금·토·일)"` |

## 4. coverage.label 생성 룰

JSON에 들어가는 `coverage.label` 한국어 문자열 생성 룰:

- **같은 달, 단일 일자**: `"5월 25일 (월)"`
- **같은 달, 연속 일자**: `"5월 22~25일 (금·토·일·월)"`
- **달이 바뀐 연속 일자**: `"2월 28일~3월 2일 (토·일·월)"`
- **이산 일자**(드물지만): 가운데에 휴일이 끼어 일자 묶음이 분리되면 `"5월 25일·27일"` 형태

요일 표기는 한 글자 (`월·화·수·목·금·토·일`).

## 5. 사용자 안내 멘트

기간 결정 결과는 반드시 사용자에게 한 줄로 알린다.

- 평일 표준: `"어제(5/26 화) 보도를 정리하겠습니다."`
- 월요일: `"오늘은 월요일이라 5월 22~25일(금·토·일) 묶어서 정리하겠습니다."`
- 공휴일 직후: `"5월 23~25일이 부처님오신날 연휴라 5월 22일 금요일까지 묶어서 보겠습니다."`
- 발행 스킵: `"오늘(5/25)은 부처님오신날 대체공휴일이라 발행을 건너뜁니다. 5/26(화)에 다시 실행해 주세요."`

## 6. dayOfWeek 매핑

JSON 출력의 `dayOfWeek` 필드 (오늘 요일):

| JS getDay() | dayOfWeek |
|---|---|
| 0 | 일 |
| 1 | 월 |
| 2 | 화 |
| 3 | 수 |
| 4 | 목 |
| 5 | 금 |
| 6 | 토 |

## 7. 왜 이렇게 분기하나

- **공휴일 처리**: 한국 매체는 휴일에 거의 안 돌아간다. 단일 일자만 보면 후보가 부족하고 자칫 글로벌 일변도로 흐른다. 묶음 처리로 다양성·신선도를 유지.
- **월요일 3일 묶음**: 토·일은 한국 보도가 없으므로 금요일까지 함께 보아야 자연스러움.
- **공휴일 자동 인식**: 사용자가 매번 공휴일을 알려주지 않아도 되도록 정적 JSON으로 박아 둠.
