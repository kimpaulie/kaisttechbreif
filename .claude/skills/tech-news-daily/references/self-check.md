# 자체 검수 체크리스트 (F6)

SKILL.md `[7] 자체 검수` 단계의 10개 항목. 모두 통과해야 `[8] JSON 작성`으로 넘어간다.

## 체크리스트

각 항목을 명시적으로 확인하고, 미통과가 하나라도 있으면 해당 카드를 재작성한 뒤 처음부터 다시 검수한다.

| # | 항목 | 통과 기준 |
|---|---|---|
| 1 | 최신성 (엄격) | 모든 사건의 **1차 발생일** 또는 **새 사실 추가 후속 보도일**이 `coverage.from` ~ `coverage.to` 범위 안에 있음. 단순 재정리 기사는 인정 안 함 (`curation-rules.md` §3 참조) |
| 2 | 제목 구체성 | 모든 제목에 **회사명/기관명**과 **핵심 숫자** 둘 다 포함 |
| 3 | 본문 3문장 | 모든 `summary`가 정확히 3문장. 마침표 개수 또는 종결어미 개수로 카운트 |
| 4 | 글머리표 없음 | `summary`에 `•`, `-`, `*`, 번호 매기기, 줄바꿈이 없음 |
| 5 | 자극어 회피 | "충격", "역대급", "대박", "미쳤다", "엄청난", "굉장한" 등 미사용 |
| 6 | AI 정형어 회피 | "결론적으로", "요컨대", "다음과 같이", "~라고 합니다" 미사용 |
| 7 | 달러 원화 환산 | 달러 금액 언급 시 모두 원화 환산 병기 |
| 8 | 한국 단독 이슈 비중 | 한국 기업·산업 단독 이슈가 최소 3개 |
| 9 | **개인 아티클·블로그 카드** (필수) | 개인 뉴스레터·블로그·인사이드 분석 글이 **최소 1개** 포함 (`curation-rules.md` §2-1-1). 언론사 스트레이트 보도만으로 10개 채우면 탈락 |
| 10 | 첫 카드 검증 | 1번 카드가 후보 풀 중 가장 큰 자본 시그널 |

## 추가 형식 검증 (스키마 단)

JSON 파일을 쓰기 직전, 다음을 자동 검증:

- `newsItems.length === 10`
- 모든 `order`가 1~10이고 중복 없음, 배열 인덱스 순서와 일치
- 모든 `sources` 배열이 비어 있지 않음, 모든 원소에 `name`/`url` 존재
- 모든 `url`이 `https://`로 시작
- `id`가 `YYYY-MM-DD` 정규식과 일치
- `dayOfWeek`가 `{월,화,수,목,금,토,일}` 중 하나

## 미통과 처리 절차

1. 어떤 항목이 어떤 카드에서 실패했는지 메모.
2. 해당 카드만 재작성한다 (전체 다시 X).
3. 재작성 후 처음부터 10개 체크 + 형식 검증 다시 돌린다.
4. 동일 카드가 2회 연속 미통과면 후보 풀로 돌아가 다른 후보로 교체.
5. 후보 풀이 부족하면 `references/curation-rules.md` 섹션 6(후보 부족 시 처리)을 따른다.

## 검수 로그 (선택)

검수 결과는 사용자에게는 보여주지 않지만, 디버깅을 위해 메모리에서 다음을 추적하면 도움이 된다:

```
재작성 횟수: N회
가장 자주 실패한 항목: #3 (본문 3문장)
교체된 카드: 03번 → 07번
```

10개 모두 통과 후 `[8] JSON 작성`으로 진행.
