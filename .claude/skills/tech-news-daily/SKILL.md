# tech-news-daily

KAIST 경영대학 테크 네트워크 **데일리 테크 브리프** 홈페이지(`/Users/cheil/dev/technews/`)에 매일 한 호분의 데이터를 자동으로 채워 넣고 GitHub에 푸시까지 완료하는 스킬.

## 원칙

이 스킬은 다음 4가지를 한 번에 자동화한다.

1. **큐레이션**: 시간 컨텍스트, 다중 검색, 4차원 평가, 다양성 필터로 10건 선정
2. **글쓰기**: 친절한 존댓말 3문장 산문, 자극어/AI 정형어 회피, 달러는 원화 환산 병기
3. **JSON 작성**: 홈페이지가 기대하는 스키마(`data/YYYY-MM-DD.json`)로 저장
4. **퍼블리시**: `git add` + `commit` + `push` 자동 실행

## 사용 시점 (트리거)

사용자가 다음과 같이 말하면 자동으로 작동:
- "어제 테크 뉴스 정리해줘"
- "오늘의 테크 브리프 만들어줘"
- "데일리 브리프 업데이트해줘"
- "이번 주말 핫한 IT 이슈 알려줘"
- "테크 브리프 발행해줘"
- "어제 테크 정리"

## 실행 프로토콜 (10단계)

각 단계의 상세 규칙은 `references/` 와 `personas/` 의 보조 파일을 참조한다. 이 SKILL.md는 단일 진실 공급원(SOT)이며, 보조 파일들이 SKILL.md를 다시 서술하지는 않는다.

### [0] 입력 파싱

- 사용자 발화에서 **기간**을 추출한다. 명시 없으면 "어제".
- 사용자가 페르소나를 명시했는지 확인한다. 명시 없으면 디폴트(카이스트 MBA).

### [1] 시간 컨텍스트 확정 → `references/time-context.md`

- KST(UTC+9) 기준 발행 일자/요일 결정.
- **평일(화~금)**: 어제 단일 일자.
- **월요일**: 금/토/일 3일 묶음.
- **공휴일 전후**: 휴일 전체 + 직전 평일까지 묶음.
- 사용자에게 한 줄 안내 (예: `"어제(5/24 일요일)는 부처님오신날이라 보도가 적어 22~24일 묶어서 정리하겠습니다."`).

### [2] 페르소나 적용 → `personas/kaist-mba.md`

- 디폴트는 카이스트 MBA 결.
- 한국 비중 4 / 글로벌 6 가중치. 자본 시그널 비중 ↑.

### [3] 다중 web_search → `references/search-strategy.md`

- 카테고리 A(글로벌 자본/딜), B(한국 산업/기업), C(화제 아티클), D(신서비스/오픈소스) 모두 커버.
- **총 6~10회** 호출. 적으면 후보 부족, 많으면 토큰 낭비.
- 한국어 매체 보도가 있으면 `sources` 배열의 앞쪽에 배치하지만, 없어도 영문 단독으로 후보 가능.

### [5] 큐레이션 → `references/curation-rules.md`

- 4차원 점수(대화 가치, 자본 시그널, 한국 함의, 신선도): 합산 12점 미만 탈락.
- 다양성 슬롯 배분: 글로벌 자본 2~3 / 한국 단독 3~4 / 인수·딜 1 / 신플레이어 1 / 아티클·OSS 1~2 / 표준·생태계 0~1.
- 최신성 필터: 발행일이 지정 기간 밖이면 무조건 탈락. (후속 보도가 기간 내 있으면 OK.)
- 정치/사회 색채 강한 이슈는 제외 (기술 본질 흐림).
- 같은 회사·같은 사건 중복 제거.

### [6] 작성 → `references/writing-style.md`

- 제목: 한글 22~32자, 회사명·핵심 숫자 포함.
- 본문: **정확히 3문장**, "~습니다/입니다" 존댓말 산문, 글머리표 절대 금지.
- 달러는 원화 환산 병기, 큰 숫자는 한국인 체감 단위 비교.
- 금지어(자극어, AI 정형어) 회피.

### [7] 자체 검수 → `references/self-check.md`

- 11개 체크리스트 모두 통과해야 한다.
- 미통과 항목이 있으면 해당 카드를 재작성한 뒤 재검수. 통과할 때까지 반복.

### [8] JSON 파일 작성 → `references/schema.md`

다음 위치에 신 포맷(`sources` 배열)으로 저장:

- 경로: `/Users/cheil/dev/technews/data/<id>.json`
- `id`: 발행 일자 (`YYYY-MM-DD`)
- `volumeNumber`: `data/` 폴더의 최신 일자 파일 안의 `volumeNumber + 1`. 파일이 하나도 없으면 `1`.
- `dayOfWeek`: 한 글자 (`월`, `화`, `수`, `목`, `금`, `토`, `일`).
- `publishedAt`: `<id>T06:30:00+09:00` 형태 ISO.
- `newsItems`: 정확히 10개. 각 항목은 `order`(1~10), `title`, `summary`(3문장), `sources`(배열) 필드를 가진다. `sources`의 각 원소는 `name`/`url` 필드.
- 한국어 매체를 `sources` 배열의 **앞쪽**에 배치한다. 영문 원문은 뒤쪽 보조.

**동일 일자 파일이 이미 존재**: 사용자에게 덮어쓸지 한 번 묻는다 (`AskUserQuestion`). 단, 사용자 발화에 "덮어써", "강제로", "그냥 덮어" 같은 표현이 있으면 묻지 않고 진행.

### [9] Git 자동화

다음을 순차 실행:

```bash
cd /Users/cheil/dev/technews
git add data/<id>.json
git commit -m "Add daily brief: <id>"
git push
```

- 각 단계의 실패는 즉시 사용자에게 보고한다. **파일은 이미 저장된 상태**이므로 사용자가 수동으로 후속 작업을 할 수 있도록 명확히 안내.
- 원격이 없거나 인증이 실패하면 push만 실패하고 commit은 남는다. 그 상태를 정확히 알린다.

## 출력 형식 (사용자 메시지)

스킬이 완료되면 사용자에게 다음 형식으로 한 번에 보고:

```
✅ 데일리 테크 브리프 발행 완료
- 일자: 2026-05-26 (화요일)
- Vol. 88
- 파일: data/2026-05-26.json
- Git: pushed to origin/main

오늘의 10건:
01 [헤드라인]
02 [헤드라인]
...
10 [헤드라인]
```

## 절대 규칙 (어기지 말 것)

1. **본문은 정확히 3문장**이어야 한다. 2문장도, 4문장도 안 된다.
2. **글머리표 사용 금지**. 줄글 산문만.
3. **자극어 사용 금지**: "충격", "역대급", "대박", "미쳤다", "엄청난", "굉장한".
4. **AI 정형어 금지**: "결론적으로", "요컨대", "다음과 같이", "~라고 합니다".
5. **달러 표기는 항상 원화 환산 병기**.
6. **JSON 스키마 위반 금지** — 홈페이지가 그대로 렌더링하기 때문에 필드 누락/오타가 곧 사이트 깨짐.
7. **사용자가 명시적으로 멈추라고 하지 않는 한 git push까지 자동 진행**.

## 엣지 케이스

- **`data/`에 파일이 0개**: `volumeNumber = 1`로 시작.
- **후보가 10개 미만**: web_search를 1~2회 추가 시도. 그래도 부족하면 사용자에게 솔직히 보고하고 가능한 만큼만 발행할지 묻는다.
- **공휴일 자동 판별 불가**: 사용자가 발화에서 알려준 공휴일 정보를 우선 사용. 모르면 일반 평일 룰로 진행.
- **이미 푸시된 일자 재발행**: 새 commit으로 덮어쓴다. 사용자가 force push를 요청하지 않는 한 force 옵션은 쓰지 않는다.
- **git이 초기화되지 않은 경우**: 파일 저장만 성공하고 git 단계는 건너뛰면서 사용자에게 안내.
