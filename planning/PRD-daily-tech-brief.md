# KAIST 경영대학 테크 네트워크 - 데일리 테크 브리프 Product Requirements Document

---

## 1. 개요

### 1.1 제품 정의

**데일리 테크 브리프**는 KAIST 경영대학 테크 네트워크가 매일 평일 오전에 발행하는 글로벌 테크 뉴스 큐레이션 서비스입니다. MBA 학생과 동문이 출근길 3분 안에 그날의 핵심 테크 흐름 10가지를 파악할 수 있도록 설계됩니다.

### 2.1 주 사용자

KAIST 경영대학 MBA 재학생 및 동문. 30~40대, 임원·창업가·전략직무 종사자. 시간이 부족하지만 테크 트렌드 파악이 업무에 직결되는 사람들.

## 3. 제품 요구사항

### 3.1 페이지 구조

페이지는 세 개의 주요 섹션으로 구성됩니다.

**3.1.1 헤더 (Hero)**

KAIST 브랜드 블루(#1f4899) 배경의 풀블리드 영역. 카이스트 로고 + "KAIST 경영대학 테크 네트워크" 텍스트가 한 줄에 배치되고, 그 아래에 큰 제호 "데일리 테크 브리프"가 위치합니다.

**3.1.2 오늘의 브리프**

날짜 바(예: 2026.05.25 월)가 큼지막하게 표시되고, 그 아래에 10개의 뉴스가 세로 리스트로 나열됩니다. 각 뉴스는 번호·제목·아웃링크 아이콘·펼침 아이콘으로 구성됩니다.

**3.1.3 지난 호 보기 (Archive)**

날짜별로 지난 호를 클릭할 수 있는 리스트. 각 행은 "날짜 · 그날의 대표 헤드라인 · 화살표" 구조이며, 행 전체가 클릭 가능합니다. 하단에 "전체 아카이브 보기" 버튼 배치.

### 3.2 핵심 기능 요구사항

**FR-01. 뉴스 카드 펼침/접힘**

뉴스 카드의 제목 영역을 클릭하면 카드 하단에 상세 요약이 펼쳐집니다. 한 번에 하나의 카드만 펼쳐지며, 다른 카드를 클릭하면 이전 카드는 자동으로 접힙니다. 펼침/접힘은 부드러운 트랜지션(약 0.4초)으로 동작해야 합니다.

**FR-02. 원문 링크**

각 뉴스 카드의 우측에는 아웃링크 아이콘이 있으며, 클릭 시 새 창(`target="_blank"`)으로 원문 출처 페이지가 열립니다. 펼쳐진 상세 카드 내부에도 "○○○에서 원문 읽기" 버튼이 별도로 존재합니다.

**FR-03. 상세 요약 포맷**

펼쳐진 상세 카드의 요약은 반드시 다음 규칙을 따릅니다.

- 정확히 3개의 문장으로 구성
- 모든 문장은 존댓말(~합니다, ~예상됩니다, ~밝혔습니다 등)로 종결
- 줄글 형식이며 글머리표 사용 금지
- 첫 문장: 사건의 핵심 사실
- 둘째 문장: 배경 또는 세부 정보
- 셋째 문장: 비즈니스적 함의 또는 한국 관련성

**FR-04. 아카이브 네비게이션**

지난 호 영역의 각 날짜 행은 클릭 가능하며, 클릭 시 해당 날짜의 브리프 페이지(`/archive/YYYY-MM-DD` 형태)로 이동합니다. 최근 7~8호를 기본 노출하고, "전체 아카이브 보기" 버튼으로 별도 아카이브 페이지로 진입할 수 있습니다.

**FR-05. 반응형 레이아웃**

데스크탑(1024px 이상), 태블릿(720px~1023px), 모바일(719px 이하) 세 가지 브레이크포인트를 지원합니다. 모바일에서는 번호·제목·아이콘 크기가 자동 축소되며 펼쳐진 상세 카드의 좌측 들여쓰기가 제거됩니다.

### 3.3 콘텐츠 운영 요구사항

매일 평일 오전 6시 30분 발행을 목표로 합니다. 편집자는 발행 전날 저녁 또는 당일 새벽에 10개의 뉴스를 선정·작성하며, 각 뉴스는 다음 필드를 채워야 합니다.

| 필드        | 형식         | 제약                                 |
| ----------- | ------------ | ------------------------------------ |
| 번호        | 자동 (01~10) | 발행 우선순위 = 번호 순서            |
| 제목        | 텍스트       | 한글 기준 20~28자 권장               |
| 요약 (상세) | 텍스트       | 정확히 3문장, 존댓말                 |
| 출처명      | 텍스트       | 영문 정식 명칭 (예: Financial Times) |
| 원문 URL    | URL          | https:// 프로토콜 필수               |

### 4.3 타이포그래피

프리젠테이션 슬라이드처럼 깔끔하고 가독성 높은 산세리프 위주 시스템.

**4.3.1 웹폰트**

```html
<!-- Google Fonts -->
<link
  href="https://fonts.googleapis.com/css2?
  family=Pretendard:wght@400;500;600;700;800;900&
  family=Inter:wght@400;500;600;700;800&
  display=swap"
  rel="stylesheet"
/>
```

대안으로 Pretendard 공식 CDN 사용 가능:

```html
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard/dist/web/static/pretendard.min.css"
/>
```

**4.3.2 폰트 스택**

| 용도                  | 폰트       | 비고                                             |
| --------------------- | ---------- | ------------------------------------------------ |
| 본문·제목 (한글 우선) | Pretendard | 한글·영문·숫자 모두 최적화된 한 가지 폰트로 통일 |
| 숫자·영문 강조        | Inter      | 큰 숫자(번호, 날짜)에 한해 보조 사용 가능        |

> Pretendard 한 가지로 통일하는 것을 1순위로 권장합니다. 폰트 종류가 많을수록 프리젠테이션 느낌이 흐트러집니다.

**4.3.3 타이포 스케일**

| 토큰           | 크기    | 굵기 | 자간     | 용도                        |
| -------------- | ------- | ---- | -------- | --------------------------- |
| Display        | 64~88px | 900  | -0.035em | 제호 "데일리 테크 브리프"   |
| Date           | 40~64px | 800  | -0.03em  | 오늘 날짜                   |
| Section Title  | 32~44px | 800  | -0.025em | "지난 호 보기" 등 섹션 제목 |
| News Number    | 44px    | 800  | -0.03em  | 뉴스 번호 (01~10)           |
| News Title     | 20~28px | 700  | -0.02em  | 뉴스 제목                   |
| Detail Summary | 18px    | 400  | -0.005em | 펼쳐진 상세 요약            |
| Body           | 17px    | 400  | 0        | 일반 본문                   |
| Meta           | 14~15px | 500  | 0        | 부가 정보 (최소화)          |
| Caption        | 13px    | 500  | 0        | 푸터, 작은 라벨             |

**원칙: 13px 미만 폰트 사용 금지.** 잔글씨를 최소화하여 프리젠테이션 슬라이드와 같은 가독성을 확보합니다.

| 분석                  | Plausible 또는 Google Analytics 4 | —                         |

### 5.2 데이터 모델

```typescript
// 발행 단위
type Edition = {
  id: string; // 예: "2026-05-25"
  volumeNumber: number; // 87
  publishedAt: ISODateString;
  dayOfWeek: "월" | "화" | "수" | "목" | "금" | "토" | "일";
  newsItems: NewsItem[]; // 정확히 10개
};

// 개별 뉴스 항목
type NewsItem = {
  order: number; // 1~10
  title: string; // 한글 제목
  summary: string; // 정확히 3문장 존댓말
  sourceName: string; // "Financial Times"
  sourceUrl: string; // https:// 원문 URL
};
```

### 5.3 라우팅

```

/archive/[date]              특정 날짜의 브리프 (예: /archive/2026-05-25)
/subscribe                   구독 신청 페이지
/about                       서비스 소개
```

| 지표                           | 목표                 |
| ------------------------------ | -------------------- |
| Lighthouse Performance         | 95점 이상            |
| LCP (Largest Contentful Paint) | 1.5초 이하           |
| CLS (Cumulative Layout Shift)  | 0.05 이하            |
| 첫 페이지 JS 번들              | 100KB 이하 (gzipped) |

### 5.5 접근성 (Accessibility)

- 모든 인터랙티브 요소는 키보드로 접근 가능해야 합니다.
- 펼침/접힘 토글은 `aria-expanded` 속성을 정확히 반영합니다.
- 아이콘 버튼은 `aria-label` 또는 `title` 속성을 가집니다.
- 본문 텍스트 컨트라스트는 WCAG AA 기준(4.5:1) 이상을 만족합니다.

## 6. 콘텐츠 가이드라인

### 6.1 뉴스 선정 기준

매일 10개의 뉴스를 선정할 때 다음 4가지 카테고리에서 균형을 맞춥니다.

| 카테고리           | 권장 비중 | 예시                                             |
| ------------------ | --------- | ------------------------------------------------ |
| 글로벌 빅테크      | 30%       | Apple, Google, Microsoft, Meta, Amazon, Nvidia   |
| AI · 반도체        | 30%       | OpenAI, Anthropic, 삼성·SK 반도체, 파운드리 동향 |
| 한국 · 아시아 테크 | 25%       | 네이버, 카카오, 쿠팡, 한·중·일 스타트업          |
| 시장 · 정책 · 자본 | 15%       | IPO, M&A, 규제(AI Act, 반독점), VC 동향          |

### 6.2 제목 작성 규칙

| 규칙                            | 예시                               |
| ------------------------------- | ---------------------------------- |
| 한글 우선, 고유명사는 원문 유지 | "Nvidia, 시가총액 5조 달러 돌파"   |
| 숫자는 아라비아 숫자 사용       | "삼성 2nm 양산 6개월 앞당겨"       |
| 한글 기준 20~28자               | 너무 길지도 짧지도 않게            |
| 형용사·부사 최소화              | "엄청난" "굉장한" 같은 주관어 금지 |

### 6.3 3줄 요약 작성 규칙

세 문장의 역할이 명확해야 합니다.

| 문장  | 역할                       | 예시                                                |
| ----- | -------------------------- | --------------------------------------------------- |
| 1문장 | What — 핵심 사실           | 엔비디아가 시가총액 5조 달러를 돌파하였습니다.      |
| 2문장 | Why / How — 배경·세부      | 메타·MS·구글의 데이터센터 발주가 직접적 원인입니다. |
| 3문장 | So What — 함의·한국 관련성 | 국내 HBM 공급망에도 즉각적인 파급이 예상됩니다.     |

모든 문장은 존댓말 종결어미(~합니다, ~하였습니다, ~예상됩니다, ~밝혔습니다)로 마무리합니다.

## 7. 출시 로드맵

### 7.1 마일스톤

| 단계                         | 기간 | 산출물                                |
| ---------------------------- | ---- | ------------------------------------- |
| Phase 0 — 기획 확정          | 1주  | PRD 최종, 디자인 시안 확정            |

| Phase 2 — 콘텐츠 운영 시스템 | 2주  | Notion/Sanity 연동, 자동 빌드         |

| Phase 4 — 분석 · 개선        | 상시 | 사용자 행동 분석, A/B 테스트          |

### 7.2 MVP 범위 (Phase 1)

다음 기능만 포함합니다.

- 오늘의 브리프 페이지 (10개 뉴스 + 펼침/접힘)
- 지난 호 보기 (최근 7~8호)
- 개별 아카이브 페이지
- 반응형 레이아웃
- 원문 링크

다음 기능은 제외합니다.

- 검색
- 카테고리 필터
- 사용자 계정
- 댓글
- 다국어 (한국어만 우선 제공)
