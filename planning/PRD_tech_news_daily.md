# PRD: 어제의 테크 뉴스 큐레이션 스킬
## (tech-news-daily)

**작성일**: 2026년 5월 25일
**작성자**: 주인님 + Claude (8차 반복 학습 결과)
**버전**: v1.0
**상태**: 초안 (개발 착수 전 리뷰 필요)

## 1. 한 줄 요약 (Elevator Pitch)

> 한국 테크 종사자가 동료/네트워크와 대화 소재로 쓸 만한 "어제의 핫한 테크 뉴스 10개"를, 한국어 보도가 있는 사건만 골라, 친절한 존댓말 산문 형식으로 큐레이션해주는 Claude 스킬.

---

## 2. 문제 정의 (Why)

### 2.1 사용자의 문제
- 매일 아침 테크 뉴스를 직접 읽기엔 시간이 부족하다
- 일반 뉴스 앱은 정치/연예/스포츠가 섞여 시그널 대비 노이즈가 높다
- 영문 매체(TechCrunch, Bloomberg)만 보면 한국 시장 함의가 빠진다
- 국내 매체만 보면 글로벌 자본 흐름·인사이더 디테일이 늦거나 빠진다
- 카드뉴스 형태의 SNS 콘텐츠는 너무 캐주얼해서 비즈니스 대화 소재로 부족하다

### 2.2 해결하지 못한 기존 도구의 한계
- **언론사 앱**: 큐레이션이 약하고 페르소나 반영 안 됨
- **AI 뉴스레터(스낵, 그린다 등)**: 일방향, 사용자 페르소나 반영 약함
- **단순 web_search 호출**: 최신성/한국 함의/타입 다양성 같은 휴리스틱이 없어서 결과가 들쑥날쑥

### 2.3 왜 Claude 스킬인가
- web_search로 실시간 데이터 확보 가능
- 큐레이션 휴리스틱을 프롬프트에 박을 수 있음
- 페르소나별 옵션 분기 가능
- 출력 톤(존댓말 산문 / 카드뉴스 / 슬랙 메시지 등) 유연하게 변환 가능

## 3. 타깃 사용자 (Who)

### 3.1 1차 페르소나: 카이스트 MBA 테크 네트워크
- 카이스트 경영대학원 재학생
- 동기/교수/동문이 개발자·PM·창업자·VC·대기업 전략직군
- 본인이 학내 네트워크 미팅에서 화두를 던지는 역할
- 시간은 부족하지만 깊이는 원함
- **이 페르소나가 디폴트**

### 3.2 2차 페르소나 (확장 가능성)
- 스타트업 창업자/공동창업자
- VC 심사역, 액셀러레이터
- 대기업 IT/전략/기획 부서
- 테크 리포터, 뉴스레터 운영자

### 3.3 페르소나에 따라 달라지는 것
- 한국 비중 (MBA는 한국 4 / 글로벌 6, 글로벌 VC는 한국 2 / 글로벌 8 등)
- 화제 결 (창업자는 신서비스/오픈소스 비중 ↑, MBA는 자본 시그널 비중 ↑)

## 4. 핵심 사용 시나리오 (How it's used)

### 4.1 트리거 발화
사용자가 다음과 같이 말할 때 스킬이 작동:
- "어제 테크 뉴스 정리해줘"
- "오늘의 테크 브리핑 만들어줘"
- "이번 주말 핫한 IT 이슈 알려줘"
- "내가 볼만한 어제 뉴스 골라줘"
- "테크 카드뉴스 원고 줘봐"

### 4.2 표준 워크플로우

```
[1] 시간 컨텍스트 확정 (user_time_v0)
    ├─ "어제"의 실제 날짜 계산
    ├─ 요일 확인 (주말/공휴일 여부)
    └─ 기간 결정: 평일 1일 | 주말 묶음 3일 | 월요일 = 금토일

[2] 페르소나 확인
    ├─ 사용자가 페르소나 명시했으면 그것 사용
    ├─ 안 했으면 디폴트(카이스트 MBA 결) 사용
    └─ 명시적 변경 요청 시 동적 조정

[3] 다중 검색 (6~10회 web_search)
    ├─ A. 글로벌 자본/딜 시그널
    ├─ B. 한국 산업/기업
    ├─ C. 화제 아티클·블로그
    └─ D. 신서비스·오픈소스

[4] 한국어 보도 검증 (필수)
    └─ 각 후보 항목에 대해 한국어 보도 존재 여부 확인
        - 한국 보도 없으면 → 탈락
        - 영문만 있으면 → 탈락

[5] 큐레이션 판단 (4차원 평가 + 다양성)
    ├─ 대화 가치 / 자본 시그널 / 한국 함의 / 신선도
    └─ 최종 10개 선정, 콘텐츠 타입 다양성 보장

[6] 작성 (정해진 포맷)
    ├─ 제목: "N ｜ [헤드라인]"
    ├─ 본문: 3문장 친절한 존댓말 산문
    ├─ 출처: 한국어 매체 우선, 영문은 보조
    └─ 순서: 큰 자본 → 한국 이슈 → 신서비스 → 아티클

[7] 자체 검수
    └─ 체크리스트 통과 후 출력
```

### 4.3 출력 형식

```markdown
# 📰 어제의 고르고 고른 테크 뉴스
**발행일**: YYYY년 MM월 DD일 | **다룬 기간**: MM월 DD일~DD일

---

## 01 ｜ [한 줄 헤드라인]

[3문장 친절한 존댓말 산문]

🇰🇷 [한국어 출처1] · [한국어 출처2]
🇺🇸 [영문 출처] (있을 때만)

---

(02~10 동일 반복)

---
```

## 5. 핵심 기능 명세 (What)

### 5.1 [F1] 시간 컨텍스트 자동 판단

**입력**: 사용자 발화 (예: "어제", "지난 주말", "최근")
**처리**:
- user_time_v0로 현재 시간 확보
- 요일 분기:
  - 평일 (화~금): 어제 단일 날짜
  - 월요일: 금/토/일 3일 묶음
  - 주말 직후 + 공휴일 끼면: 휴일 전체 + 직전 평일까지

**출력**: 사용자에게 한 줄 안내
> "어제(5/24 일요일) 단독은 부처님오신날이라 뉴스가 적어, 22~24일 묶어서 정리하겠습니다."

**왜 필요한가**: v7~v8에서 발견된 가장 큰 빈틈. 일요일/공휴일에는 한국 매체가 거의 안 돌아간다.

---

### 5.2 [F2] 다중 검색 전략

**검색 카테고리 (반드시 모두 커버)**:

| 카테고리 | 횟수 | 쿼리 예시 |
|---|---|---|
| A. 글로벌 자본/딜 | 2~3 | "AI funding round [날짜]", "tech M&A [날짜]" |
| B. 한국 산업/기업 | 2~3 | "삼성/SK하이닉스 [키워드]", "한국 AI 스타트업 [날짜]" |
| C. 화제 아티클·블로그 | 1~2 | "와우테일 AI [주제]", "브런치 [키워드]" |
| D. 신서비스·오픈소스 | 1 | "GitHub trending", "Product Hunt this week" |

**총 6~10회**. 너무 적으면 후보가 부족하고, 너무 많으면 토큰/시간 낭비.

**한국어 보도 검증 추가 검색**:
- 영문 원문이 잡혔으면 → "[기업명] 한국 보도" 추가 검색
- 한국어로 못 잡으면 후보에서 탈락

---

### 5.3 [F3] 큐레이션 판단 로직

**1차: 4차원 점수**
- 대화 가치 (1~5): 동료에게 던질 수 있는가
- 자본·전략 시그널 (1~5): 숫자/구조 변화의 의미
- 한국 함의 (1~5): 한국 기업/산업에 영향
- 신선도 (1~5): 일반 흐름에서 비껴난 디테일

**합산 12점 미만 탈락**

**2차: 다양성 필터** (10개 슬롯 배분)
- 글로벌 자본 시그널: 2~3개
- 한국 단독 이슈: 3~4개 (1차 페르소나 가중치)
- 인수/딜 드라마: 1개
- 새 플레이어·서비스: 1개
- 화제 아티클/오픈소스: 1~2개
- 표준·생태계 전환: 0~1개

**필터 (절대 규칙)**:
- ❌ 한국어 보도 없는 항목
- ❌ 같은 회사·같은 사건 중복
- ❌ 정치/사회 색채 강한 이슈 (기술 본질 흐림)
- ❌ 3일 이상 지난 사건 (단, 후속 보도가 기간 내 있으면 OK)

---

### 5.4 [F4] 글쓰기 문체 규칙

**제목**
- 형식: `## NN ｜ [한 줄 헤드라인]`
- 길이: 22~32자
- 회사명·핵심 숫자 포함
- ✅ "앤트로픽, 다음 주 1,349조원 밸류로 펀딩 마감 임박"
- ❌ "AI 회사가 큰 투자를 받았다" (구체성 없음)

**본문: 3문장 산문**
- 불렛/번호 절대 금지
- "~습니다/입니다" 존댓말
- 1문장: 사건 사실
- 2문장: 핵심 숫자·디테일
- 3문장: 의미·함의

**숫자 표기**
- 달러는 원화 환산 병기: "900억 달러(약 1,349조 원)"
- 큰 숫자는 한국인 체감 가능 단위로: "삼성전자 시총과 비슷"

**금지어**
- 자극어: "충격", "역대급", "대박", "미쳤다"
- AI 정형어: "결론적으로", "요컨대", "다음과 같이", "~라고 합니다"

**출처 표기**
- 🇰🇷 한국어 보도가 메인 (1~3개)
- 🇺🇸 영문 원문은 보조 (있을 때만)
- 형식: `[매체명 "기사 핵심"](URL)`

---

### 5.5 [F5] 순서 배치 룰

- 1번: 가장 큰 자본 시그널
- 2번: 두 번째 큰 시그널 (글로벌)
- 3~5번: 인수·한국 핫이슈 교차
- 6~8번: 신플레이어·한국 기업
- 9~10번: 아티클·오픈소스로 가볍게 마무리

**리듬**: 큰 거 → 작은 거 → 큰 거 → 작은 거
**국적**: 글로벌-한국 교차 배치

---

### 5.6 [F6] 자체 검수 체크리스트 (출력 직전)

- [ ] 10개 모두 한국어 보도 1개 이상 있는가
- [ ] 발행일이 지정 기간 안에 있는가 (최신성 룰)
- [ ] 제목에 회사명/숫자가 있는가
- [ ] 본문이 정확히 3문장인가
- [ ] 본문에 불렛이 없는가
- [ ] 자극어가 없는가
- [ ] 달러는 원화 환산 병기됐는가
- [ ] 한국 단독 이슈가 3개 이상인가
- [ ] 아티클/오픈소스 카드가 최소 1개 있는가
- [ ] 첫 카드가 가장 큰 자본 시그널인가
- [ ] 출처 링크가 클릭 가능한 마크다운 형식인가

체크리스트 미통과 항목 발견 시 → 해당 카드 재작성 후 재검수.

## 6. 시행착오에서 배운 핵심 교훈 (v1~v8 회고)

### v1: 단순 뉴스 나열
- **문제**: 정치/사회 이슈가 섞임. 톤이 잡담스러움.
- **교훈**: 페르소나를 처음부터 분명히 해야 함.

### v2: 기술 중심
- **문제**: 너무 잔잔하고 학술적. 화제성 부족.
- **교훈**: "기술"이 아니라 "기술이 만드는 변화"가 화두.

### v3: 큰 이야기 (의미 중심)
- **문제**: 의미 해석은 좋지만 일상감 부족.
- **교훈**: 자본 시그널이 가장 잘 통하는 결.

### v4: SNS/일상 결
- **문제**: 너무 가벼움. MBA 페르소나와 안 맞음.
- **교훈**: 페르소나 맞춤이 결정적.

### v5: MBA 자본 시그널 (6장)
- **문제**: 흥미가 한쪽으로 쏠림. 신선한 거 부족.
- **교훈**: 자본+신서비스+오픈소스+아티클 균형 필요.

### v6: 자본+신서비스+오픈소스 균형 (8장)
- **문제**: 한국 보도가 없는 항목 섞임. 출처 비대칭.
- **교훈**: **한국어 보도 검증을 필수 게이트로**.

### v7: 한국어 보도 검증 추가 (10장)
- **문제**: 한국 보도 없는 항목에 "한국어: 국내 보도 적음" 라벨 달아둠 → 사용자 짜증.
- **교훈**: **한국 보도 없으면 라벨도 달지 말고 그 항목 자체를 빼라**.

### v8: 최신성 룰 누락
- **문제**: 한 달 전 사건도 섞임. "지난 토일월만" 룰을 안 박음.
- **교훈**: **최신성 룰을 명시적 필터로**. 발행일이 지정 기간 밖이면 무조건 탈락.

### 통합 교훈
1. **하드 룰을 명시적으로 박을 것** — "유연하게"는 모호함의 시작
2. **출력 톤은 페르소나가 결정** — 결을 먼저 합의해야 함
3. **한국어 보도가 있는지가 결정적 게이트** — 한국 사용자에게 영문만 던지면 가치 0
4. **공휴일·주말 처리는 별도 분기** — 어제가 일요일이면 룰을 바꿔야 함

## 7. 비기능 요구사항

### 7.1 성능
- 한 번의 호출에 web_search 6~10회로 제한
- 총 응답 시간: 60~120초 내외
- 토큰 사용: 합리적 범위 (정확한 한계는 측정 후 결정)

### 7.2 신뢰성
- 한국어 보도 검증 게이트가 작동하지 않으면 출력 거부
- 최신성 룰 미충족 항목은 자동 탈락
- 출처 URL은 클릭 가능한 마크다운 형식 보장

### 7.3 확장성
- 페르소나 추가 가능한 구조 (현재 1개 + 확장 슬롯)
- 출력 포맷 추가 가능 (산문 / 카드뉴스 PNG / 슬랙 메시지 등)
- 검색 카테고리 추가 가능

### 7.4 한계 명시
- Threads, X(트위터), Instagram 본문은 봇 차단으로 접근 불가
- 이런 소스를 활용해야 할 때는 "기사화·블로그화된 2차 소스" 기반임을 사용자에게 명시
- 1차 소스 직접 접근이 필수인 경우는 솔직히 한계 고지
